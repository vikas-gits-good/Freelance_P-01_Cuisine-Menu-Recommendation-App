<GRAPHDB_PROMPT>

4. Always return variables with names.
5. Always limit the data retrieved from the query in the range of 1 to 200.
6. Use only labels and relationships that are defined in graph schema, do not create new ones. 
7. Do not edit the graph. Do not change data and do not delete data.

Example Query:-
query = """
MATCH (rstn:Restaurant {ids: 418})-[link:HAS_MENU]->(menu:Menu)
RETURN
    rstn.name AS rstn_name, // specific variable as name
    rstn.area AS rstn_area,
    menu.name AS menu_name,
    menu.types AS menu_types,
    link.price AS menu_price,
    link.rating AS menu_rating
ORDER BY menu_rating DESC, menu_name ASC // ordering data
LIMIT 100 // limiting data
"""
